⚠️ Identified Pain Points in AS-IS System

Manual Data Entry — Everywhere

Vehicle No, consignee, broker, GSTN all entered manually at every step. Zero pre-fill from master data. High error rate across 65 branches.

No Real-Time Party Verification

PAN, GSTIN, Bank Account of transporters/vendors entered by staff — no API verification at point of entry. Fraud risk & compliance gaps.

POD is Paper-Based Scan

Driver physically returns POD → scan at office → manual upload. 3–7 day lag before billing can start. No mobile capture at delivery point.

Zero Client Communication Integration

No automated alerts to clients for CN booking, in-transit status, delivery, or bill submission. All communication is ad hoc phone/email.

Bill Submission is Manual & Untracked

Physical bill courier to client. Submission date entered manually later. No email tracking. Delayed entries mean DSO calculations are wrong.

No Single Source of Truth

Invoice PDFs in email, POD images in WhatsApp, rate charts in Excel, client outstanding in separate ledger software — no unified data layer.

HM Detention Calculated Manually

Detention days, delay penalties, unloading charges all computed manually in FM Slip. Errors and disputes with transporters are frequent.

e-Way Bill & e-Invoice Disconnected

e-Way Bill number entered manually into TMS from a separate portal. No API link. e-Invoice (IRN) generation tracked separately.

Single Source of Truth Architecture
All data flows into one system. Every module writes to and reads from the same verified, real-time data layer — no silos, no reconciliation nightmares.

PAN Verification

Validate PAN format, fetch legal name and PAN status.

→ FM/Hire Slip creation · Vendor Master

GSTIN Verification

Pull legal name, trade name, address, nature of business, turnover slab, constitution, filing status.

→ Client Master · Vendor Master · Bill Creation GST Validation

CIN Search

Fetch incorporation details, directors, company status from MCA. For corporate clients.

→ Client Core ID · Legal entity verification

PAN to CIN

Derive CIN from PAN where CIN not directly provided by party.

→ Client & Vendor Master auto-resolve

PAN to MSME

Auto-fetch Udyam number from PAN if entity is MSME-registered.

→ Vendor Master · MSME payment compliance

Udyam Verification

Verify Udyam registration and pull enterprise category (Micro/Small/Medium).

→ Transporter onboarding · MSME tier badge

IEC Verification

Validate IEC registration for importers/exporters. Conditional for JNPT/port clients.

→ JNPT CN Creation · Export client onboarding

Bank Account Verification

Verify account number + IFSC, fetch bank name, branch, account holder name.

→ Vendor Bank KYC · Balance Payment before NEFT

Mobile to Account

Link mobile number to bank account/UPI VPA; validate name match for fraud prevention.

→ Vendor Bank fallback · Driver payment verification

Mobile Name Lookup

Fetch subscriber name from mobile number — validates driver/transporter identity.

→ Driver Contact KYC · FM Hire Slip verification
